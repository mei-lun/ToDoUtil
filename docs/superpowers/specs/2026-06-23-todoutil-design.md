**日期**：2026-06-23
**状态**：设计阶段（需求与架构已与用户确认）

---

## 1. 产品定位

一个钉在桌面的极窄竖条 ToDo 工具，每日打开能立即看到"今天要做什么"，做完即勾掉，未完成自动累积到明天，归档后通过时间线可视化每个任务"从计划到完成用了多久"。

**核心使用场景**：

- 每天早上打开窗口，看今天该做的事
- 突发想法通过全局快捷键秒级记录
- 任务可推迟、可搁置（暂时不追）、可放弃
- 周末翻归档，看自己最近完成了哪些事、哪些拖了很久

**非目标**：

- 不是团队协作工具（纯单人）
- 不是闹钟（时间仅作元信息显示，不弹通知）
- 不是 GTD 全功能管理器（无标签、无项目、无优先级）

## 2. 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 桌面框架 | Electron | 跨平台、UI 美观可定制度高 |
| 前端 | React + Vite + TypeScript | 组件化清晰、生态成熟、类型安全 |
| 状态管理 | Zustand | 轻量、API 简单 |
| 存储 | JSON 文件（分三份） | 可读、零依赖；数据量上限后可平滑迁 SQLite |
| Markdown | marked + DOMPurify | 行业标准、防 XSS |
| 日志 | electron-log | Electron 生态标准 |
| 进程模型 | 主/渲染严格分离，IPC 通信 | Electron 安全最佳实践 |

## 3. 数据模型

### 3.1 Task

```ts
type Task = {
  id: string                    // uuid v4
  title: string                 // 剥离 @ 部分后的纯标题
  rawTitle: string              // 原始输入（含 @），编辑时显示
  detail: string                // markdown 源码
  plannedDate: string           // YYYY-MM-DD，无 @ 则=创建日
  plannedTime?: string          // HH:mm，仅元信息显示，不通知
  createdAt: string             // ISO 时间戳
  status: 'active' | 'done' | 'abandoned' | 'pooled'
  doneAt?: string               // status 转 done/abandoned 时的时间戳
  order: number                 // 同日内手动排序权重，越小越靠前
  attachments: string[]         // 图片相对路径数组
  originalPlannedDate: string   // 首次计划日，永不变；用时起点
  pooledRanges: Array<{from: string, to: string}>  // 历次搁置区间
}
```

**字段说明**：

- `plannedDate` 会随"推迟"或"0:00 自动滚动"变化；`originalPlannedDate` 永不变，用于归档时计算用时
- `pooledRanges` 记录每次进出搁置池的日期对，用于"用时计算时减去搁置期"
- `status` 单一字段判定任务所属视图，避免多个 boolean 互斥状态打架

### 3.2 三个 JSON 文件的分工

| 文件 | 存什么 | 迁移规则 |
|---|---|---|
| `tasks.json` | `status=active` 的任务 | 创建 / 推迟 / 拉回搁置 → 进；完成、放弃、搁置 → 出 |
| `pool.json` | `status=pooled` 的任务 | 搁置时进；拉回某天时出 |
| `archive.json` | `status=done` 或 `abandoned` 的任务 | 仅追加，永不修改 |

### 3.3 Config

```ts
type Config = {
  alwaysOnTop: boolean           // 默认 true
  immersiveMode: boolean         // V2 沉浸模式，MVP 留字段，禁用
  shortcut: string               // 默认 "CommandOrControl+Alt+T"
  windowBounds: {x, y, width, height}
  archiveGridBounds?: {width, height}  // 网格视图记忆的宽度
  dataDir: string                // 默认 userData，可改成 OneDrive 路径
  notificationsEnabled: boolean  // 永久 false（MVP 不做通知）
  backupRetentionDays: number    // 默认 30
}
```

存 `config.json`。

## 5. UI 设计

### 5.1 风格定位

macOS / Apple 风：圆角、半透明毛玻璃、克制灰阶、留白和呼吸感。参考 Things 3 / macOS 提醒事项。

### 5.2 主窗口（默认 280×600）

**无标题栏**：去掉系统标题栏（`frame: false`），**常态下没有任何拖动条**——窗口的全部空间都是内容。

**常态（未输入）**：

```
┌──────────────────────────────────┐
│ ＋                               │ 左上角加号，点开才展输入框
├──────────────────────────────────┤
│  ◀  今天 6/23  ▶      📥3  📚    │ 日期切换 + 搁置(3) + 归档
├──────────────────────────────────┤
│ ○ ▓ 写需求文档           9:30   │
│ ○   买牛奶                       │
│ ○ ▓▓▓ 修登录 bug                │ 延期 3 天
│ ─────────                        │
│ ● 整理桌面                       │ 已完成沉底灰化
│ ● ~~买水~~                       │ 已放弃
└──────────────────────────────────┘
```

**输入展开态（点 ＋ 后）**：横向展开为输入框，自动获焦

```
┌──────────────────────────────────┐
│ ＋ 输入任务，回车添加…       Esc │ ← 自动展开+获焦
├──────────────────────────────────┤
│  ◀  今天 6/23  ▶      📥3  📚    │
└──────────────────────────────────┘
```

**输入框收起规则**：
- 回车提交 → 添加任务，输入框清空并**收回 ＋ 按钮**
- 按 `Esc` → 放弃输入，收回
- 点输入框外部（失焦）且内容为空 → 收回
- 失焦但有内容 → 保留展开态（防止误失焦丢内容）

**移动模式**（仅在需要重新摆放窗口时进入）：

- 入口：托盘右键菜单 `↔ 移动窗口`
- 进入后：
  - 整个窗口背景变成淡蓝半透明覆盖层（明确"现在是移动态"）
  - 鼠标移到窗口任意位置 → 变成 ✥ 移动光标
  - 整个窗口任意位置都可按住拖动（`-webkit-app-region: drag` 覆盖整窗）
  - **期间禁止编辑/点击**：所有任务行、输入框、按钮临时不响应
- 退出：
  - 拖动一次松手 → 自动退出移动模式（背景覆盖层淡出）
  - 按 `Esc` → 也可主动退出

**无标题栏后的功能替代**：

| 原标题栏元素 | 新方案 |
|---|---|
| 拖动窗口 | 托盘右键 `↔ 移动窗口` 进入移动模式，全窗可拖，拖完自动退出 |
| 📌 置顶切换 | 托盘右键菜单 `✓ 钉在桌面`（勾表示已开启） |
| ⚙ 设置 | 托盘右键菜单 `⚙ 设置` |
| ─ 最小化 | 取消（窗口隐藏靠 Esc 或托盘） |
| ✕ 关闭 | 取消（窗口隐藏靠 Esc 或托盘） |
| 隐藏窗口 | 按 `Esc`（无展开/编辑状态时）或点托盘图标 |

### 5.3 任务行交互

- **常态**：左侧圆圈 `○` + 可选延期色块 `▓` + 标题 + 右侧可选时间标注
- **Hover**：右侧浮出 `→ 📥 ✎ ✕` 四图标
  - `→`：推迟到明天
  - `📥`：进搁置池
  - `✎`：展开详情并切到编辑模式
  - `✕`：标记放弃
- **单击行（非图标区）**：行内展开详情区
- **同时只展开一项**：点新的会自动收起旧的
- **点 `○`**：勾选完成（动画 0.2s → `●`），2 秒缓冲期内再点可撤销，2 秒后才迁入 archive.json
- **长按 0.3s**：进入拖拽排序态
- **已完成任务**：沉到列表底部，灰化显示
- **已放弃任务**：归档里灰+删除线，不在今日视图

### 5.4 详情区

```
┌──────────────────────────────┐
│ 渲染后的 markdown            │
│ ☐ 子任务 1                   │ 渲染态下可勾，立即写回源码
│ ☑ 子任务 2                   │
│ [图]                         │
│                              │
│              ✎ 编辑          │ 右上角
└──────────────────────────────┘
```

- **默认渲染态**：展示 markdown 渲染结果
- **编辑模式**：点 ✎ → 切换到纯 `<textarea>` 源码（无分屏预览）
  - 失焦或 `Ctrl+S` 保存
  - `Esc` 放弃编辑（弹确认）
- **子任务**：渲染态可点勾，主任务行右上角显示 `N/M` 进度
  - **不联动**：所有子任务勾完不会自动完成主任务，主任务永远手动点圆圈

### 5.5 顶部输入框

- **常态**：左上角一个 ＋ 按钮，不占空间
- **点 ＋ 或 按热键** → 横向展开成输入框，自动获焦
- 回车 = 添加任务（plannedDate 默认 = 当前视图所看的日期），添加后自动清空并收回
- `Esc` = 放弃输入，收回
- 失焦时：内容为空则收回，有内容则保留展开态
- 输入时实时高亮 `@` token（让你知道哪些会被识别）

### 5.6 全局快速添加浮层

- 全局快捷键 `Ctrl+Alt+T`（默认，可改）触发
- 屏幕中央浮出极小输入框，自动获焦
- `Enter` 添加并淡出关闭，`Esc` 取消
- 不打开/不切到主窗口

## 6. 核心交互流程

### 6.1 创建任务

1. 用户输入文字（顶部输入框 / 全局快捷键浮层 / 托盘快速添加）
2. 按回车
3. `date-parser` 扫描 `@` 开头的 token
4. 命中 → 提取 `plannedDate` / `plannedTime`，从 `rawTitle` 剥离得到 `title`
5. 未命中 → `plannedDate = 今天`，`title = rawTitle`
6. 生成 Task，`status=active`，`order = 该日期已有任务数+1`（追加底部）
7. 写入 `tasks.json`
8. IPC 通知所有窗口刷新

### 6.2 `@` 解析白名单

| 类型 | 示例 |
|---|---|
| 相对日期 | `@今天 @明天 @后天 @大后天` |
| N 天后 | `@三天后 @5天后` |
| 日期 | `@6月25日 @6.25 @6-25 @6/25` |
| 星期 | `@周一~@周日 @下周一~@下周日` |
| 时间 | `@9点 @9:30 @上午9点 @下午3点 @晚上8点` |
| 组合 | `@明天9点 开会` |

**规则**：
- 没有 `@` 前缀 → 完全不识别
- 日期无年份 → 默认本年；如本年该日已过 → 视为明年
- 多个 `@` token → 后者覆盖前者
- 识别错 → 用户改标题重写 / 点日期 chip 弹小日历手动改

### 6.3 勾选完成

1. 用户点 `○`
2. `○ → ●` 动画 0.2s
3. `status = 'done'`，`doneAt = now`，`order = -1`（沉底）
4. **2 秒缓冲**期内再点 `●` → 撤销完成，回到 active
5. 2 秒后从 `tasks.json` 迁到 `archive.json`
6. IPC 刷新

### 6.4 0:00 自动滚动

启动时 `setTimeout` 到下一个 0:00 触发；同时 `setInterval` 每分钟检查兜底（防止机器睡眠跳过 0:00）。

触发逻辑：

1. 扫描 `tasks.json` 中所有 `status=active` 且 `plannedDate < 今天` 的任务
2. 每个任务 `plannedDate = 今天`（`originalPlannedDate` 不变）
3. 写回 `tasks.json`，IPC 通知渲染
4. 重新调度下一个 0:00

### 6.5 延期色块计算

```
延期天数 = 今天 - originalPlannedDate
0   → 无色块
1-2 → ▓   浅红 #FCA5A5
3-5 → ▓▓  中红 #EF4444
6-10 → ▓▓▓ 深红 #B91C1C
>10 → ▓▓▓▓ 暗红 #7F1D1D
```

### 6.6 推迟 / 搁置 / 放弃

| 操作 | 数据变化 | 文件迁移 |
|---|---|---|
| **→明天** | `plannedDate += 1` | tasks.json 内变动 |
| **📥搁置** | `status='pooled'`，`pooledRanges` 追加 `{from: 今天, to: null}` | tasks.json → pool.json |
| **✕弃** | `status='abandoned'`，`doneAt=now` | tasks.json → archive.json |

### 6.7 从搁置池拉回

1. PoolView 点任务 → "拉回到..." 按钮
2. 弹小日历选日期（默认今天）
3. `status='active'`，`plannedDate = 选定日`
4. `pooledRanges` 最后一项的 `to = 今天`
5. pool.json → tasks.json

### 6.8 用时计算（归档）

```
用时(天) = (doneAt 的日期 - originalPlannedDate)
         - sum(每个 pooledRange 的天数)
         + 1   // 同一天创建并完成 = 1 天
```

## 7. Markdown / 附件

### 7.1 Markdown 渲染

- 库：`marked` + `DOMPurify`
- 支持：标题、列表、加粗/斜体、行内 code / 代码块、链接、图片、引用、表格、子任务 checkbox
- 不支持：HTML 直插（DOMPurify 过滤）、数学公式、mermaid

### 7.2 编辑模式

- 点 ✎ → `<textarea>` 纯源码编辑（无分屏预览）
- 失焦 / `Ctrl+S` → 保存切回渲染
- `Esc` → 弹确认放弃编辑

### 7.3 子任务

- 源码 `- [ ]` / `- [x]` 标准 markdown 语法
- 渲染态可点勾，立即写回源码并保存
- 主任务行右上角显示 `N/M` 进度
- **不联动**：全部勾完不自动标完成；主任务永远手动点圆圈

### 7.4 图片处理

**粘贴流程**（编辑态 Ctrl+V）：
1. 渲染进程拿剪贴板二进制
2. IPC 传主进程
3. 主进程 sha256 hash → 用 hash 命名
4. 存到 `userData/attachments/<task-id>/<hash>.png`
5. 返回相对路径，编辑器自动插入 `![](attachments/<task-id>/<hash>.png)`

**拖拽支持**：从文件管理器拖图到详情区 → 同样流程。

**URL 图片**：`![](https://...)` 仍支持，但不复制到本地。

**清理策略**：
- 任务永久删除 → 同步删 `attachments/<task-id>/` 整目录
- 编辑时删图片引用 → **不立即删文件**（防误删）
- 后台每 30 天扫一次：遍历所有任务的 markdown，找出未被引用的孤儿 → 移到 `attachments/.trash/`，再 7 天后真删

### 8.1 窗口管理

|---|---|

| 隐藏 | 隐藏到托盘，不退出进程 |
| 显示 | 左键托盘图标切换显示/隐藏 |
| 置顶切换 | 托盘菜单 `✓ 钉在桌面`，调 `setAlwaysOnTop()`，菜单项打勾反映状态 |
| 拖出屏幕 | 至少 50px 留屏内 |
| 显示器丢失 | 重置到主屏中央 |

### 8.2 系统托盘

```
托盘右键：
  ＋ 快速添加任务...
  ─────────
  显示主窗口
  ↔ 移动窗口          ← 进入移动模式
  ✓ 钉在桌面          ← 勾表示当前已置顶，点击切换
  ─────────
  📥 搁置池 (3)
  📚 归档
  ⚙ 设置
  ─────────
  关于
  退出
```

左键单击 = 切换主窗口显示/隐藏。

**Esc 键作用域**（优先级从高到低）：
- 在移动模式中 → 退出移动模式
- 输入框展开中 → 收回输入框
- 任务详情编辑中 → 退出编辑（弹确认未保存）
- 任务详情展开中 → 收起详情
- 以上都不在 → 隐藏窗口到托盘

### 8.3 全局快捷键

- 默认 `Ctrl+Alt+T`，可在设置改（录制式）
- 注册失败 → 设置面板红字提示，不阻塞启动

### 8.4 数据目录变更

- 选新目录 → 检查 `tasks.json` 是否存在
  - 无 → 复制当前数据 → 切换指向
  - 有 → 弹三选一："用新目录数据 / 用当前覆盖新目录 / 取消"
- 切换后强制重启应用

## 9. 备份与错误处理

### 9.1 自动备份

```
每天首次启动：
  if 今天还没备份：
    复制 tasks.json   → backups/YYYY-MM-DD-tasks.json
    复制 pool.json    → backups/YYYY-MM-DD-pool.json
    复制 archive.json → backups/YYYY-MM-DD-archive.json
  清理 > 30 天的备份
```

- **附件不备份**（体积太大）
- **无 GUI 恢复按钮**：手动到备份目录拷文件覆盖

### 9.2 错误处理

| 场景 | 处理 |
|---|---|
| JSON 读取失败（损坏） | 尝试加载最近备份，弹横幅"已从 X 日备份恢复，原文件存到 .corrupted/" |
| JSON 写入失败 | toast 红字，操作回滚，永不静默丢数据 |
| 写入崩溃保护 | 原子写：`tasks.json.tmp` → `rename` |
| 自动备份失败 | 重试 1 次，仍失败 → toast 提示磁盘空间 |
| 图片复制失败 | toast 提示，详情区不插入引用，编辑内容保留 |
| Markdown 渲染异常 | 退回纯文本，console 报错，不让一条坏数据炸整个详情 |
| 0:00 跨时区 | 用本地时间，跨时区返回后下次启动一次性补滚 |
| 快捷键注册失败 | 设置面板红字，不阻塞启动 |

### 9.3 日志

- `electron-log`，路径 `userData/logs/main.log`
- 单文件 5MB，保留 5 个
- 默认 info，崩溃自动升 debug
- 设置面板"打开日志目录"按钮

## 11. 风险与开放问题

1. **JSON 性能上限**：估算 5000+ 条任务后读写会变慢、并发写易损坏。需要在 `tasks-repo` 抽象层留好接口，未来迁 SQLite 不影响上层逻辑。
2. **0:00 滚动鲁棒性**：用户笔记本睡眠/合盖跨过 0:00 是常见场景，每分钟兜底是必要的；测试需覆盖"机器从睡眠唤醒"。
3. **附件目录膨胀**：长期使用后 attachments 目录可能数 GB。提供"打开附件目录"按钮让用户自查。
4. **沉浸模式 V2 的预留**：MVP 在 Config 留字段 + 设置面板灰显，UI 层 window-manager 设计要预留"无边框+透明+鼠标穿透切换"的扩展点。
